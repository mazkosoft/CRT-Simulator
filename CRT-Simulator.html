这里负责 JSON 导入导出、本地保存，以及内置 / 用户预设管理。

MEDIA & EXPORT / 媒体与导出

图片也可以导出成视频，会按当前动态闪烁、光带、VHS 抖动录制这段时长。

视频质量主要受码率限制。建议 1080p 用 30–60Mbps，2K 以上用 60–120Mbps。

导出时临时提高 WebGL 后台渲染分辨率。越高越清晰，但越慢。

图片导出视频时会在后台录制动态 CRT/VHS 效果；视频导出时会逐帧处理当前视频。

这版支持图片和视频。媒体、RGB、点遮罩、光带先合成，再一起桶形变形；光带等待阶段完全隐藏，扫过时是下方平直亮边、上方渐变拖影。

视频默认静音循环播放。播放时每帧会更新到 WebGL 纹理并进入同一套 CRT 管线。

像素块大小自动锁定 RGB / Cell Period，因此会和 RGB 荧光粉、点遮罩分辨率保持一致。

作用在辉光叠加后的最终画面上。0 是黑白，1 是原始饱和度，2 以上会更电子屏。

图片可直接导出 PNG；视频会把当前 WebGL 处理后的画面实时录制并导出为 WebM。

CRT CORE / CRT 核心
PHOSPHOR & MASK / 荧光粉与遮罩

RGB 周期和点遮罩周期完全一致，并且会一起参与桶形变形。

BEAM & GLOW / 光带与辉光

辉光现在是先对完整 CRT 画面离屏模糊，再用 screen 叠加。

VHS LAB / VHS 实验室

这一组负责录像带化处理。色度相关参数现在全部并入 VHS：其中 NTSC Chroma Bleed 会按亮度 / 色度分离思路，只压缩色度带宽，不是简单 RGB 硬分离。

依据 NTSC / VHS 的亮度-色度分离思路，对色度分量做横向低通与带宽压缩，不是简单 RGB 偏移。

模拟低价/老旧设备常见的过度边缘增强，在黑色边缘右侧产生白色拖影。

控制边缘增强和振铃强度。需要明显效果时建议 2.0–4.0。

控制白色振铃光晕宽度。数值越高越像粗糙、廉价的模拟电路。

正值让红色与蓝色向相反水平方向分离；负值会反向。

这不是硬边 RGB 偏移，而是用更宽的多点柔化采样分离红/蓝通道，再重新合成。

这一组是录像带化重点：NTSC 色度渗色、家用录像机锐化、隔行、切换噪声、掉磁横线都在这里。
